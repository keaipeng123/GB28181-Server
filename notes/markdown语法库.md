# 我的 Markdown 语法库

> 目标：快速写出结构清晰、可发布的 Markdown 文档。

---

## 1. 标题（Heading）

### 1.1 ATX 风格（推荐）

使用`#`，可表示1-6级标题。
> \# 一级标题   
> \## 二级标题   
> \### 三级标题   
> \#### 四级标题   
> \##### 五级标题   
> \###### 六级标题    

效果：
> # 一级标题   
> ## 二级标题   
> ### 三级标题   
> #### 四级标题   
> ##### 五级标题   
> ###### 六级标题

### 1.2 Setext 风格（较少用）

使用`=`和`-`标记一级和二级标题。
> 一级标题   
> `=========`   
> 二级标题    
> `---------`

效果：
> 一级标题   
> =========   
> 二级标题
> ---------  

---

## 2. 段落与换行（Paragraph & Line Break）

- **段落**：段落之间用空行分隔。
- **强制换行**：行尾加 **两个空格** 再回车。

```md
第一行（末尾两个空格␠␠）  
第二行

新段落
```

---

## 3. 引用（Blockquote）

在段落的每行或者只在第一行使用符号`>`,还可使用多个嵌套引用，如：
> \> 区块引用  
> \>> 嵌套引用  

效果：
> 区块引用  
>> 嵌套引用

---

## 4. 代码（Code）

### 4.1 行内代码

```md
用 `ctrl+a` 作为示例。
```

### 4.2 代码块（推荐：围栏代码块）

代码区块的建立是在每行加上4个空格或者一个制表符（如同写代码一样）。如    
普通段落：

void main()    
{    
    printf("Hello, Markdown.");    
}    

代码区块：

    void main()
    {
        printf("Hello, Markdown.");
    }

**注意**:需要和普通段落之间存在空行。

---

## 5. 强调（Emphasis）

在强调内容两侧分别加上`*`或者`_`，如：
> \*斜体\*，\_斜体\_    
> \*\*粗体\*\*，\_\_粗体\_\_

效果：
> *斜体*，_斜体_    
> **粗体**，__粗体__

---

## 6. 列表（List）

### 6.1 无序列表

```md
- 第一项
- 第二项
- 第三项

+ 也可以用加号
* 也可以用星号
```

### 6.2 有序列表

```md
1. 第一项
2. 第二项
3. 第三项
```

- 提示：标记后面至少要有一个空格。

---

## 7. 分割线（Horizontal Rule）

```md
---

***

___
```

---

## 8. 链接（Link）

### 8.1 行内链接

```md
[链接文字](https://example.com "可选标题")
```

效果：

[链接文字](https://example.com "可选标题")


### 8.2 参考式链接（适合长文复用）

```md
[链接文字][id]

[id]: https://example.com "可选标题"
```

效果：

[链接文字][id]

[id]: https://example.com "可选标题"

注意：`[id]: ...` 这类定义通常放在段落末尾或文末，不要放在引用块里（否则可能被当作引用内容）。

---

## 9. 图片（Image）

```md
![图片描述](https://example.com/image.png "可选标题")
```

- 图片语法和链接几乎一样，只是前面多一个 `!`。

效果：

![图片描述](https://example.com/image.png "可选标题")

---

## 10. 转义（Escape）

当你想把 Markdown 的“语法符号”当成普通字符显示时，用反斜杠 `\` 转义。

```md
\* 这不是列表
\# 这不是标题
\> 这不是引用
\` 这不是行内代码标记
```

---

## 11. 表格（扩展语法，非传统 Markdown）

多数平台（如 GitHub）支持表格：

```md
| 名称 | 链接 |
| --- | --- |
| Markdown | https://example.com |
```

对齐方式：

```md
| 左对齐 | 居中 | 右对齐 |
| :--- | :---: | ---: |
| a | b | c |
```

---

## 12. 我的写作小习惯（实用约定）

- 标题层级尽量不要跳级（比如 `#` 后直接用 `###`）。
- 列表项中要写多段内容时，建议空行 + 缩进对齐。
- 文档较长时优先使用参考式链接，统一放在文末维护。

---

## 13. 兼容性提醒

不同 Markdown 解释器/平台对“扩展语法”（如表格、脚注、任务列表等）的支持不完全一致；写文档前先确认目标平台的渲染规则。
